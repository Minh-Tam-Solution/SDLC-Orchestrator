# FR-053: Bootstrap Files Loader

**Version**: 2.0.0
**Status**: APPROVED
**Created**: March 2026
**Sprint**: 218
**Framework**: SDLC 6.1.1
**Epic**: EP-07 Multi-Agent Team Engine — MTClaw Restructure
**ADR**: ADR-056 (LD#1 Snapshot Precedence), ADR-058 (ZeroClaw Best Practices)
**Owner**: Backend Team
**Pattern Source**: MTClaw `bootstrap/files.go` + 8 templates

---

## 1. Overview

### 1.1 Purpose

Load agent personality and context from 6 workspace-resident markdown files at conversation creation time. Files are read once and frozen into the conversation snapshot per ADR-056 LD#1 (snapshot immutability). Subagent sessions receive only AGENTS.md + TOOLS.md to save ~40% tokens. When workspace files do not exist, the system falls back to the DB `system_prompt` column. TEAM.md and AVAILABILITY.md are VIRTUAL files generated by the FR-051 context injector, NOT on-disk bootstrap files.

### 1.2 Problem Being Solved

| Before FR-053 | After FR-053 |
|--------------|-------------|
| Agent personality is a single `system_prompt` column | 6 composable files, each with a clear responsibility |
| Prompt changes require DB update + admin UI | Edit a markdown file in workspace, next conversation picks it up |
| No workspace path resolution standard | 4-level SSOT: conversation -> team -> project -> default |
| Subagents receive full context (wasteful tokens) | FilterForSession: subagents get AGENTS.md + TOOLS.md only (~40% savings) |
| No file-based fallback when DB prompt exists | Graceful fallback: files found -> use files; no files -> use DB system_prompt |
| TEAM.md and AVAILABILITY.md confused with on-disk files | VIRTUAL files clearly documented as generated (FR-051), not on-disk |

### 1.3 Business Value

- **Developer experience**: Edit markdown files in workspace instead of navigating admin UI
- **Token efficiency**: Subagent sessions skip 4 of 6 files (~40% token savings)
- **Composability**: Teams can customize IDENTITY.md without touching TOOLS.md
- **Snapshot immutability**: Files frozen at conversation start — mid-conversation edits do not affect running sessions (ADR-056 LD#1)

### 1.4 Scope Boundary

| In scope | Out of scope |
|----------|-------------|
| 6 bootstrap files: AGENTS.md, IDENTITY.md, TOOLS.md, MEMORY.md, HEARTBEAT.md, DELEGATION.md | SOUL.md, BOOTSTRAP.md, USER.md (deferred — CTO Correction 2: 6 files ONLY) |
| Workspace path SSOT resolution (4 levels) | File watching / hot-reload during conversation |
| FilterForSession at conversation creation time (CTO Correction 4) | Dynamic re-filtering during conversation |
| Fallback to DB system_prompt when files missing | Migration tool to convert DB prompts to files |
| VIRTUAL file documentation (TEAM.md, AVAILABILITY.md) | VIRTUAL file generation (covered by FR-051) |

### 1.5 CTO Corrections Applied

| Correction | Detail |
|-----------|--------|
| CTO Correction 2 | 6 files ONLY (not 11). SOUL.md, BOOTSTRAP.md, USER.md are nice-to-have, deferred. |
| CTO Correction 4 | FilterForSession at conversation CREATION TIME, not query time (ADR-056 LD#1 snapshot). |
| Consolidated M1 | Workspace path SSOT resolution chain consolidated into single function. |

## 2. Functional Requirements

### 2.1 Bootstrap File Loading

#### FR-053-01: Load all 6 files from workspace directory

```gherkin
Scenario: All 6 files present in workspace directory
  GIVEN workspace directory "/projects/bflow/" contains:
      AGENTS.md, IDENTITY.md, TOOLS.md, MEMORY.md, HEARTBEAT.md, DELEGATION.md
  WHEN bootstrap_loader.load(workspace_path="/projects/bflow/", agent_name="coder") is called
  THEN all 6 files are read from disk
  AND their contents are returned as a dict:
      {"AGENTS": "<content>", "IDENTITY": "<content>", "TOOLS": "<content>",
       "MEMORY": "<content>", "HEARTBEAT": "<content>", "DELEGATION": "<content>"}
  AND each file content is stripped of leading/trailing whitespace
  AND bootstrap_found = true is set
```

```gherkin
Scenario: Partial files present
  GIVEN workspace directory "/projects/bflow/" contains only:
      AGENTS.md, TOOLS.md
  WHEN bootstrap_loader.load(workspace_path="/projects/bflow/", agent_name="coder") is called
  THEN present files are read: AGENTS.md, TOOLS.md
  AND missing files return empty string: IDENTITY="", MEMORY="", HEARTBEAT="", DELEGATION=""
  AND no error is raised (missing files are normal)
  AND bootstrap_found = true (at least one file exists)
```

### 2.2 Fallback to DB system_prompt

#### FR-053-02: Fallback when workspace files missing

```gherkin
Scenario: No bootstrap files found — fallback to DB
  GIVEN workspace directory "/projects/bflow/" contains none of the 6 bootstrap files
  WHEN bootstrap_loader.load(workspace_path="/projects/bflow/", agent_name="coder") is called
  THEN all 6 keys return empty string
  AND bootstrap_found = false is set
  AND the caller falls back to agent_definitions.system_prompt from DB
```

```gherkin
Scenario: Workspace directory does not exist
  GIVEN workspace_path="/projects/nonexistent/" does not exist on disk
  WHEN bootstrap_loader.load is called
  THEN bootstrap_found = false is returned
  AND no exception is raised (graceful degradation)
  AND the caller falls back to DB system_prompt
```

```gherkin
Scenario: Fallback produces identical agent behavior
  GIVEN agent "coder" has system_prompt = "You are a Python code generation expert..."
  AND workspace directory has no bootstrap files
  WHEN a conversation is created for "coder"
  THEN the system_prompt from DB is used as the base prompt
  AND the conversation functions identically to a bootstrapped session
```

### 2.3 Workspace Path Resolution

#### FR-053-03: Workspace path SSOT resolution chain (4 levels)

```gherkin
Scenario: Workspace path from conversation (highest priority)
  GIVEN conversation.workspace_path = "/projects/bflow/"
  AND team.settings['workspace_path'] = "/projects/default/"
  AND project.root_path = "/projects/global/"
  WHEN resolve_workspace_path(conversation) is called
  THEN the result is "/projects/bflow/" (conversation-level wins)
```

```gherkin
Scenario: Workspace path fallback to team settings
  GIVEN conversation.workspace_path = NULL
  AND team.settings['workspace_path'] = "/projects/team-workspace/"
  AND project.root_path = "/projects/global/"
  WHEN resolve_workspace_path(conversation) is called
  THEN the result is "/projects/team-workspace/" (team-level fallback)
```

```gherkin
Scenario: Workspace path fallback to project root
  GIVEN conversation.workspace_path = NULL
  AND team.settings = {} (no workspace_path key)
  AND project.root_path = "/projects/project-root/"
  WHEN resolve_workspace_path(conversation) is called
  THEN the result is "/projects/project-root/" (project-level fallback)
```

```gherkin
Scenario: Workspace path fallback to system default
  GIVEN conversation.workspace_path = NULL
  AND team.settings = {} (no workspace_path key)
  AND project.root_path = NULL
  WHEN resolve_workspace_path(conversation) is called
  THEN the result is settings.DEFAULT_WORKSPACE_PATH (system default)
```

**Resolution order** (Consolidated M1 — first non-null wins):

```python
# Priority order:
conversation.workspace_path -> team.settings['workspace_path'] -> project.root_path -> settings.DEFAULT_WORKSPACE_PATH
```

### 2.4 FilterForSession

#### FR-053-04: Subagent session loads only AGENTS.md + TOOLS.md

```gherkin
Scenario: Normal session loads all 6 files
  GIVEN a new conversation is created with session_type = "normal"
  AND workspace contains all 6 bootstrap files
  WHEN FilterForSession(session_type="normal", bootstrap_data=<all 6>) is called
  THEN all 6 files are included in the conversation snapshot:
      AGENTS, IDENTITY, TOOLS, MEMORY, HEARTBEAT, DELEGATION
  AND the snapshot is frozen (ADR-056 LD#1 — immutable after creation)
```

```gherkin
Scenario: Subagent session loads only AGENTS + TOOLS
  GIVEN a child conversation is created with session_type = "subagent"
  AND workspace contains all 6 bootstrap files
  WHEN FilterForSession(session_type="subagent", bootstrap_data=<all 6>) is called
  THEN only 2 files are included: AGENTS, TOOLS
  AND 4 files are excluded: IDENTITY, MEMORY, HEARTBEAT, DELEGATION
  AND the excluded content is NOT stored in the snapshot (token savings ~40%)
```

#### FR-053-05: FilterForSession at creation time (snapshot immutability)

```gherkin
Scenario: Filter applied at conversation creation, not query time
  GIVEN conversation C1 is created at T1 as session_type = "subagent"
  AND FilterForSession runs at T1 and includes only AGENTS + TOOLS
  WHEN at T2 > T1, the session_type metadata could theoretically be changed
  THEN C1 still uses the T1 snapshot (AGENTS + TOOLS only)
  AND no re-filtering occurs — snapshot is immutable per ADR-056 LD#1
```

```gherkin
Scenario: Bootstrap snapshot frozen at conversation creation
  GIVEN conversation C1 was created at T1 with AGENTS.md content "Version A"
  AND at T2 > T1, the workspace AGENTS.md is edited to "Version B"
  WHEN conversation C1 continues to exchange messages at T3 > T2
  THEN C1 still uses "Version A" from its frozen snapshot
  AND a NEW conversation C2 created at T4 > T2 would use "Version B"
```

```gherkin
Scenario: Snapshot stored in conversation metadata
  GIVEN conversation C1 is created with bootstrap files loaded
  WHEN the bootstrap content is frozen
  THEN it is stored in agent_conversations.snapshot_data JSONB:
      {"bootstrap": {"AGENTS": "<content>", "TOOLS": "<content>", ...}}
  AND the snapshot_data is NOT updatable after conversation creation
```

### 2.5 Graceful Error Handling

#### FR-053-06: File not found graceful handling

```gherkin
Scenario: Individual file read error does not abort loading
  GIVEN workspace directory "/projects/bflow/" exists
  AND AGENTS.md is readable but TOOLS.md has a permission error
  WHEN bootstrap_loader.load is called
  THEN AGENTS.md content is loaded successfully
  AND TOOLS.md returns empty string with a warning log
  AND the remaining 4 files are attempted (not short-circuited)
  AND bootstrap_found = true (at least one file succeeded)
```

```gherkin
Scenario: File exceeding size limit is truncated
  GIVEN MEMORY.md is 120KB (exceeds 50KB limit)
  WHEN bootstrap_loader.load reads MEMORY.md
  THEN the content is truncated at 50KB boundary
  AND a warning is logged: "MEMORY.md truncated from 120KB to 50KB limit"
  AND the truncated content is included in the snapshot
```

## 3. Non-Functional Requirements

| NFR | Target |
|-----|--------|
| Bootstrap load latency (6 files) | < 20ms (local disk I/O) |
| Workspace path resolution latency | < 5ms (in-memory cascade) |
| FilterForSession computation | < 1ms |
| Max file size per bootstrap file | 50KB (prevent prompt explosion) |
| File encoding | UTF-8 only (reject non-UTF-8 with warning) |
| Snapshot storage overhead | <= 50KB JSONB per conversation |
| Subagent token savings | ~40% vs normal session |

---

## 4. Security Considerations

- **Path traversal prevention**: Workspace path must be validated — no `..` components allowed, must be under an allowed base directory
- **File size limit**: Each bootstrap file capped at 50KB to prevent denial-of-service via prompt explosion
- **Read-only access**: Bootstrap loader only reads files, never writes to workspace directory
- **Snapshot immutability**: `snapshot_data` is set once at conversation creation; UPDATE on this field is blocked by application-level guard (not DB constraint, for flexibility)
- **No executable content**: Bootstrap files are treated as plain text/markdown — no code execution, no template rendering
- **Permission validation**: File read operations use least-privilege filesystem access

## 5. Data Model

### 5.1 No New Tables

Bootstrap files are stored on the filesystem in the workspace directory. No new database tables are required.

### 5.2 Existing Table Usage

| Table | Column | Usage |
|-------|--------|-------|
| agent_definitions | system_prompt | Fallback when workspace bootstrap files not found |
| agent_conversations | snapshot_data (JSONB) | Stores frozen bootstrap content at conversation creation |
| agent_conversations | workspace_path | Highest-priority workspace path override |
| teams | settings (JSONB) | Contains `workspace_path` key for team-level resolution |
| projects | root_path | Project-level workspace path fallback |

### 5.3 File Responsibility Mapping

| File | Purpose | Normal session | Subagent session |
|------|---------|---------------|-----------------|
| AGENTS.md | Agent identity, role description, behavioral rules | Yes | Yes |
| IDENTITY.md | Personality traits, communication style | Yes | No |
| TOOLS.md | Available tools and usage instructions | Yes | Yes |
| MEMORY.md | Long-term context, project knowledge | Yes | No |
| HEARTBEAT.md | Current status, recent activity, sprint context | Yes | No |
| DELEGATION.md | Static delegation rules (complements FR-051 dynamic injection) | Yes | No |

### 5.4 VIRTUAL Files (NOT on-disk)

| File | Generated by | Purpose |
|------|-------------|---------|
| TEAM.md | FR-051 context injector | Team context (lead vs member variant) |
| AVAILABILITY.md | FR-051 context injector | Negative context when no delegation targets |

### 5.5 System Prompt Assembly Order

1. Base instructions (hardcoded, minimal)
2. Bootstrap files (AGENTS -> IDENTITY -> TOOLS -> MEMORY -> HEARTBEAT -> DELEGATION)
3. FR-051 dynamic context injection (TEAM.md -> DELEGATION.md -> AVAILABILITY.md)
4. FR-052 skills summary (`<available_skills>` XML block)
5. Conversation history
