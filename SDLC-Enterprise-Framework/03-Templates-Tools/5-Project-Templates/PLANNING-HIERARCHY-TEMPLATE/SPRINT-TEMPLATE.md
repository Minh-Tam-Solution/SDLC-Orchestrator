## Purpose

This template provides a standard structure for Sprint planning (5-10 days). Sprints contain committed work items and connect Phase themes to daily execution.

**Position in Hierarchy**: Roadmap → Phase → **Sprint** → Backlog

## Definition of Done

Sprint items are considered DONE when:

- [ ] Code complete and reviewed (2+ approvers)
- [ ] Unit tests passing (coverage target met)
- [ ] Integration tests passing
- [ ] Documentation updated
- [ ] Deployed to staging environment
- [ ] Product Owner acceptance

## Tier-Specific Guidelines

### LITE Tier
- **Optional** - Simple task list sufficient

### STANDARD Tier
- **Recommended** - Basic sprint tracking (goal + committed work + daily progress)

### PROFESSIONAL Tier
- **Mandatory** - Full sprint template with metrics and retro

### ENTERPRISE Tier
- **Mandatory** - Full sprint + formal review/retro documentation

---

## Checklist

Before starting a Sprint, verify:

- [ ] Sprint goal is clear and measurable
- [ ] Committed work fits team capacity
- [ ] Dependencies identified and tracked
- [ ] Definition of Done agreed upon
- [ ] Sprint events scheduled
- [ ] Team availability confirmed
