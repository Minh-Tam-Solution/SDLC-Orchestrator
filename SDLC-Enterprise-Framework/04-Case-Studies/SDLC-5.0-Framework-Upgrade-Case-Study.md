### 3. 4-Tier Classification System

```yaml
LITE (Team Size: 1-2):
  Purpose: Solo developers, hobby projects
  Requirements:
    - Design Thinking: OPTIONAL (but recommended)
    - Code Review: Self-review + checklist
    - Documentation: README + inline comments
    - Testing: Manual testing acceptable
    - Gates: G0.1, G3 only
  Time to Setup: 2 hours

STANDARD (Team Size: 3-10):
  Purpose: Startups, small product teams
  Requirements:
    - Design Thinking: MANDATORY for new features
    - Code Review: Peer review (1 reviewer minimum)
    - Documentation: ADRs + API docs
    - Testing: 80%+ unit test coverage
    - Gates: G0.1, G1, G2, G3
  Time to Setup: 1 day

PROFESSIONAL (Team Size: 10-50):
  Purpose: Scale-ups, multiple product teams
  Requirements:
    - Design Thinking: MANDATORY with CPO approval
    - Code Review: 2+ reviewers required
    - Documentation: Complete technical specs
    - Testing: 90%+ coverage + integration tests
    - Governance: Quality gates + Security gates enforced
    - Observability: Basic SLOs + alerting
  Time to Setup: 1 week

ENTERPRISE (Team Size: 50+):
  Purpose: Large organizations, regulated industries
  Requirements:
    - Design Thinking: MANDATORY with CEO review
    - Code Review: 3+ reviewers + security review
    - Documentation: Complete audit trail
    - Testing: 95%+ coverage + E2E + load tests
    - Governance: Full compliance (security, audit, change management)
    - Observability: Complete monitoring + alerting + runbooks
  Time to Setup: 2 weeks
```

